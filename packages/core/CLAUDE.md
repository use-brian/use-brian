# @sidanclaw/core

The orchestration engine. Pure TypeScript, no Express, no DB driver — stores are passed in via interfaces. **Read this first when entering this package.** For the project-wide rules (workflow, test tagging, conventions) see the root `CLAUDE.md`.

## Tests

`packages/core/src/**/__tests__/*.test.ts`. Tagged with `[COMP:area/name]`. See `docs/workflow/component-map.md` for the component → test mapping. Integration tests use `describeIf(apiKey ? describe : describe.skip)` and skip silently when `GEMINI_API_KEY` is missing.

Smoke test: `pnpm smoke` (or `pnpm --filter @sidanclaw/core smoke`). Runs `scripts/smoke.ts` — one query-loop turn against a mock provider, no DB, no network.

## Common gotchas

- **`provider.createSession()` is stateful.** Don't reuse a session across unrelated conversations — Gemini's `thoughtSignature` will leak across turns. Create a new session per `queryLoop` invocation.
- **`Tool` defaults are fail-closed.** Forgetting to set `isConcurrencySafe: true` on a read-only tool means it runs serially. Forgetting `isReadOnly: true` means the executor treats it as a write.
- **Wrapper composition order matters.** `defaultWrappers()` is the standard pipeline — don't reorder unless you know what you're doing. `wrapTextLoopPrevention` must come before `wrapIdleTimeout` so the latter catches its abort.
- **PDFs are passed natively, not extracted.** `parsers.ts` returns the raw base64 and the turn builder emits an `image`-type `ContentBlock` with `mimeType: 'application/pdf'`. Gemini handles the extraction via `inlineData`. Don't reintroduce a text-extraction library (`pdf-parse`, `pdfjs-dist`) — Gemini's native PDF reader beats local extraction for tables, scans, and layout, which is what most user uploads contain.
- **The query loop sends only tool results on subsequent turns**, not full history — the session keeps history internally. Don't manually append the assistant message back into `nextMessages`.

## When to add a new module

Match the structure of an existing one — `packages/core/src/<area>/index.ts` re-exports the public surface, `<area>/types.ts` for store interfaces, `<area>/<thing>.ts` for impl, `<area>/__tests__/<thing>.test.ts` for tests. Add `[COMP:<area>/<name>]` tags and a row in `docs/workflow/component-map.md`. Update this file's module map.
